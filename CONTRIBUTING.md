# Contributing to Web Explorer

First off, thank you for considering contributing to Web Explorer! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## üöÄ How Can I Contribute?

### Reporting Bugs
- Check the issues tab to see if the bug has already been reported.
- If not, open a new issue. Include a clear title and a detailed description with steps to reproduce the error.
- Attach any relevant logs or screenshots.

### Suggesting Enhancements
- Open an issue with the "feature request" label.
- Describe the behavior you want and why it's useful.

### Pull Requests
1. **Fork** the repository.
2. **Clone** your fork (`git clone https://github.com/your-username/web-explorer.git`).
3. **Create a branch** for your fix or feature (`git checkout -b feature/amazing-feature`).
4. **Run tests** (`npm test`) to ensure everything is working.
5. **Commit your changes** (`git commit -m 'Add amazing feature'`).
6. **Push to the branch** (`git push origin feature/amazing-feature`).
7. **Open a Pull Request**.
